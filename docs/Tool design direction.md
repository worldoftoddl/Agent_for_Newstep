# 도구 설계 방향 — Excel 탐색 도구 v2 (Phase 3.5 제안)

> 작성일: 2026-07-15 · 상태: 제안 · 선행: Phase 3 완료 · 관련: PRD F2·F4, task_list Phase 5

## 0. 전달할 방향 (한 문장)

엑셀 이해는 다섯 차원 — **값·공간·논리·시각·의도** — 의 문제다. 현재 도구 5종은
값·공간·논리 일부를 커버하지만, 감사조서에서 "수행된 절차의 흔적"은 주로
**시각(서식)과 의도(메모·숨김) 차원**에 기록된다. v2의 목표는 프롬프트가 이미
지시하는 관찰(틱마크·색상 표시·검토 서명란)을 도구가 감각기관으로 뒷받침하는 것.

## 1. 현재 상태 진단

### 유지할 것 (v1의 올바른 결정)

- **오류 = 관찰**: 예외 대신 "오류: …" 텍스트 + 복구 힌트(파일 목록, 시트 목록) 동봉
- **절단 조향**: 500셀 상한 + "범위를 나눠 요청" 안내
- **개요 선행 정책**: 프롬프트와 도구 docstring 양쪽에 명시
- **캐시값 경고**: data_only 값이 재계산이 아님을 출력·프롬프트 모두에서 고지
- **경로 한정**(`_resolve`)과 무상태 시그니처(path 매개변수) — 멀티유저 공개 데모에 적합

### 핵심 불일치 (v2의 존재 이유)

`prompts.py` [탐색 원칙]은 *"틱마크(✓, ◆ 등)·색상 표시·검토 서명란은 수행된 절차의
단서"*라고 지시하지만, **어떤 도구도 서식 정보를 반환하지 않는다.**

샘플 조서 실측(`D-10_매출채권_샘플.xlsx`):

| 시트 | bold | 배경색 | 글자색 | 문자 틱마크 |
|---|---|---|---|---|
| Lead | 7 | 5 | 28 | 0 |
| 조회확인 | 7 | 6 | 47 | ✓·◆ 3건 + 범례(A9) |
| 연령분석 | 5 | 0 | 23 | 0 |

문자 틱마크(✓)는 `excel_find`로 잡히지만, **색상으로만 표시된 마킹(시트당 글자색
23~47셀)은 에이전트에게 완전히 불가시**하다.

## 2. P0 — 서식 채널

`excel_read_range`에 `mode="format"` 추가 (기존 values/formulas와 나란히).

- 셀 값 뒤에 압축 주석 부가: `1,250,000 [B|F:FFFF00|C:0000FF]`
  (B=bold, F=배경 RGB, C=글자색 RGB). 서식 없는 셀은 주석 생략.
- 색상은 **RGB 원값**만 반환 — "노랑=검토완료" 해석은 프롬프트/에이전트 몫
  (조서마다 범례가 다르므로 도구가 의미를 확정하지 않는다).
- **구현 주의 — 색상 3계열**: openpyxl 색은 rgb/theme/indexed로 나뉘며,
  theme 색은 `.rgb`가 문자열이 아니라 접근 시 오류 객체를 반환한다.
  `fgColor.type`으로 분기해 theme는 `T{테마번호}±{tint}` 표기로 내보낸다.
  실측: 표준 서식 3650A의 음영 337셀이 전부 theme 계열(기입란 표지) —
  rgb만 처리하면 실무 표준 서식에서 즉시 깨진다.
- 프롬프트 [탐색 원칙]에 1줄 추가: *"색상·볼드 마킹 확인은 mode=format으로."*

**완료 기준**: "조회확인 시트에서 색으로 표시된 항목은 무엇인가?" 질문에
셀 주소·색상값 근거로 답변 가능. 픽스처에 색상 셀(rgb·theme 각각) 심어 단위 테스트.

## 3. P0 — 의도 채널

`excel_get_annotations(path, sheet)` 신설. 반환:

- **셀 메모**(comment): 좌표·작성자·본문 — 조서에서 메모 = 검토자 코멘트
- **숨김** 행/열/시트 목록 — 숨김에는 이유가 있다 (전기 자료, 검토 중 항목)
- **데이터 유효성** 적용 범위 — 드롭다운 = 입력 셀 표지
- **정의된 이름**(defined names) — 작성자가 직접 의미를 부여한 범위

**완료 기준**: 픽스처에 메모 3건·숨김 열 1건 심고 전량 회수. 샘플 조서에
검토자 메모 2~3건 추가해 데모 시나리오("검토자가 뭐라고 남겼어?") 확보.

## 4. P1 — 수식 지도

`excel_formula_map(path, sheet)` 신설: 시트 전체 수식을 R1C1로 정규화해
동일 패턴을 압축하고(예: 353개 → 상위 패턴 10개), **수식 지대 내 하드코딩
숫자**(같은 행·열에 수식이 존재하는 자리의 상수)를 별도 목록으로 반환.

- 조서의 검증 로직(합계 검산, tie-out, 시트간 대사) 골격을 한 호출로 파악
- 하드코딩 검출은 조서 품질 리뷰(2차 사용자: 인차지) 시나리오와 직결
- 판정은 하지 않는다 — 이탈이 오류인지 의도된 입력인지는 에이전트의 도메인 판단
- 기존 자산 재사용(N3): 검증된 참조 구현 존재 (`xlsx_agent_tools.py`의
  `formula_map` — R1C1 변환기·이탈 휴리스틱 이식, 반환만 텍스트로 변환)

**완료 기준**: 샘플 조서 3시트 각각에서 패턴 압축 출력 + 이탈 0건 이상 검출을
픽스처로 검증.

## 5. P1 — overview 테이블 감지

`excel_workbook_overview` 강화: 첫 행 미리보기(헤더가 1행이 아닌 조서에서 빗나감)
대신, **빈 행 2개 이상을 경계로 값-블록을 분할**해 시트별 블록 목록을 반환:

```
[조회확인] 3개 블록
  ① A1:H2   — 헤더부 (회사명·기간·작성자/검토자 서명란 추정)
  ② A4:H8   — 본문 테이블 (첫 행: 거래처|장부금액|회신금액|차이|틱마크…)
  ③ A9:C10  — 범례
```

블록 ref는 `excel_read_range`에 그대로 넘기는 인수가 된다 (도구 간 인계 규약).

**완료 기준**: 샘플 3시트에서 블록 경계가 사람 판단과 일치.

## 6. P2 — 잔손질 (v2 커밋에 편승)

- **워크북 캐시**: `(resolved_path, mtime, data_only)` 키 LRU(maxsize 8).
  도구 시그니처는 무상태 유지, 호출당 재로드만 제거.
- **마크다운 파이프 이스케이프**: 셀 값의 `|` → `\|` (표 깨짐 방지).
- **find 수식 검색**: `mode="formulas"` 옵션 — 시트간 참조(`=BS!…`) 추적용.
- **밀도 통계 주의**: `max_row×max_column`은 유령 셀(서식만 있는 행)로
  과대계상될 수 있음 — 실제 값 경계 기준 병기.

## 7. 반환 포맷 결정 (변경 없음, 규약만 추가)

- **현행 텍스트 반환 유지.** JSON 봉투 전환은 Phase 5 인용 표기 계층에서
  프로그램적 후처리 필요가 확정될 때 재검토 (현재는 LLM 가독성·UI 표시 겸용에
  텍스트가 우위).
- 단, 지금부터 규약화: **모든 도구 출력의 첫 줄은 출처**(`시트!범위` 또는
  파일명) — cid 인용 규칙의 엑셀판. 수치 주장에 셀 주소를 병기하는 프롬프트
  규칙과 짝을 이룬다.
